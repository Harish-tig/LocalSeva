Service Booking & Marketplace Platform API

A comprehensive RESTful API combining professional service booking with peer-to-peer marketplace functionality. Built with Django REST Framework, featuring JWT authentication, real-time status tracking, and secure data handling.

Platform Capabilities

Two integrated modules providing comprehensive service booking and marketplace functionality

01

Service Booking Module

Complete workflow management from booking creation to service completion. Providers submit quotes, customers accept offers, and both parties track real-time status updates throughout the service lifecycle.

02

Marketplace Module

Peer-to-peer product listings with category management, condition tracking, and buyer-seller communication. Direct contact exchange and comment system facilitate transparent transactions.

03

Authentication & Authorization

Secure JWT-based authentication with role-based access control. Separate permissions for customers, service providers, and marketplace sellers ensure data security and proper access levels.

04

Reviews & Safety

Comprehensive review system with star ratings and written feedback. User reporting mechanism for fraud, poor service, or safety concerns maintains platform integrity and user trust.

Authentication

POST /api/user/register/ Register new user account +
Required Fields
username string, unique, max 50 chars Required
email string, valid email, unique Required
password string, min 8 chars Required
password2 string, must match password Required
Request Example
{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "SecurePass123",
  "password2": "SecurePass123"
}
Success Response (201 Created)
{
  "message": "User registered successfully",
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user_id": 1,
  "username": "john_doe"
}
POST /api/user/login/ Authenticate and receive JWT tokens +
Required Fields
username string Required
password string Required
Request Example
{
  "username": "john_doe",
  "password": "SecurePass123"
}
Success Response (200 OK)
{
  "message": "Login successful",
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

Profile Management

GET /api/user/profile/ Retrieve current user profile +
Authentication Required
Authorization: Bearer <access_token>
Success Response (200 OK)
{
  "id": 1,
  "username": "john_doe",
  "email": "john@example.com",
  "role": "USER",
  "bio": "I need home services",
  "phone": "+1234567890",
  "location": "New York, NY",
  "is_service_provider": false,
  "rating": 0.0,
  "total_reviews": 0
}
PUT /api/user/profile/ Update user profile information +
Optional Fields (all fields are optional)
bio string
phone string
location string
avatar file
Request Example
{
  "bio": "Professional plumber with 10 years experience",
  "phone": "+1234567890",
  "location": "New York, NY"
}
POST /api/user/profile/become-provider/ Register as service provider +
Request Body
Empty - No request body required
Success Response (200 OK)
{
  "message": "You are now marked as a service provider!",
  "next_step": "Please update your profile to add service provider details",
  "required_fields": {
    "experience_years": "Required",
    "pricing_type": "Required (FIXED or FLEXIBLE)",
    "base_price": "Required (Average base/visiting charge)"
  }
}

Bookings

POST /api/user/bookings/create/ Create new service booking +
Required Fields
provider_id int - Service provider's ID Required
service_category string - e.g., "Plumbing" Required
description string - Service description Required
address string - Service address Required
scheduled_date datetime - Future date/time Required
Request Example
{
  "provider_id": 2,
  "service_category": "Plumbing",
  "description": "Leaking pipe in kitchen sink",
  "address": "123 Main St, New York, NY",
  "scheduled_date": "2023-10-15T10:00:00Z"
}
Success Response (201 Created)
{
  "id": 1,
  "user_name": "john_doe",
  "provider_name": "plumber_joe",
  "description": "Leaking pipe in kitchen sink",
  "status": "PENDING",
  "created_at": "2023-10-10T09:00:00Z"
}
GET /api/user/bookings/ List user or provider bookings +
Query Parameters
type string - "user" (default) or "provider"
status string - Filter by booking status
Success Response (200 OK)
[
  {
    "id": 1,
    "user_name": "john_doe",
    "provider_name": "plumber_joe",
    "description": "Leaking pipe in kitchen sink",
    "status": "QUOTE_GIVEN",
    "quote_price": "500.00",
    "scheduled_date": "2023-10-15T10:00:00Z"
  }
]
PUT /api/user/bookings/{id}/ Update booking status or details +
Provider: Give Quote
{
  "quote_price": "500.00",
  "provider_notes": "Will bring replacement parts"
}
Customer: Accept Quote
{
  "status": "ACCEPTED"
}
Provider: Complete Service
{
  "status": "COMPLETED",
  "final_price": 500
}
Status Flow
PENDING → QUOTE_GIVEN → ACCEPTED → IN_PROGRESS → COMPLETED

Marketplace Products

GET /api/user/marketplace/ List products with filters +
Query Parameters
category FURNITURE, ELECTRONICS, VEHICLES, etc.
condition NEW, LIKE_NEW, GOOD, FAIR, POOR
city string - Filter by city
min_price / max_price float - Price range
search string - Search title, description
Success Response (200 OK)
[
  {
    "id": 1,
    "seller_name": "john_doe",
    "title": "Sofa Set - 3 Seater",
    "category": "FURNITURE",
    "condition": "LIKE_NEW",
    "price": "15000.00",
    "city": "New York",
    "views": 45,
    "comment_count": 3
  }
]
POST /api/user/marketplace/create/ Create product listing +
Required Fields
title string - Product title Required
description string - Product description Required
category string - Product category Required
condition string - Product condition Required
price decimal - Product price Required
address string - Pickup address Required
city string - City Required
Request Example
{
  "title": "Sofa Set - 3 Seater",
  "description": "Hardly used, in excellent condition",
  "category": "FURNITURE",
  "condition": "LIKE_NEW",
  "price": "15000.00",
  "address": "123 Main St, New York",
  "city": "New York"
}
GET /api/user/marketplace/{id}/ Retrieve product details +
Note
Retrieving a product increments its view count automatically
Success Response (200 OK)
{
  "id": 1,
  "seller_name": "john_doe",
  "title": "Sofa Set - 3 Seater",
  "description": "Hardly used, in excellent condition",
  "category": "FURNITURE",
  "condition": "LIKE_NEW",
  "price": "15000.00",
  "contact_phone": "+1234567890",
  "views": 46
}
PUT /api/user/marketplace/{id}/ Update product information +
Permissions
Only the product seller can update their own products
Request Example (partial update allowed)
{
  "price": "14000.00",
  "description": "Price reduced! Hardly used sofa set"
}
DELETE /api/user/marketplace/{id}/ Remove product listing +
Note
This is a soft delete - product is deactivated, not permanently deleted
Success Response (200 OK)
{
  "message": "Product deactivated successfully"
}
GET /api/user/marketplace/my-products/ List seller's products +
Success Response (200 OK)
Returns all products created by the authenticated user

Success Response (200 OK)
[
  {
    "id": 1,
    "user_name": "buyer123",
    "comment": "Interested in this sofa. WhatsApp me at 9876543210",
    "contact_info": "WhatsApp: 9876543210",
    "created_at": "2023-10-02T10:00:00Z"

POST /api/user/marketplace/comments/create/ Add comment to product +
Required Fields
product int - Product ID Required
comment string - Comment text Required
Request Example
{
  "product": 1,
  "comment": "Interested in this sofa. WhatsApp me at 9876543210",
  "contact_info": "WhatsApp: 9876543210"
}
DELETE /api/user/marketplace/comments/{id}/delete/ Remove or hide comment +
Permissions
Comment owner: Can delete their own comment
Product seller: Can hide inappropriate comments (soft delete)
Success Response (200 OK)
{
  "message": "Comment deleted successfully"
}
// or
{
  "message": "Comment hidden successfully"
}
GET /api/user/marketplace/my-product-comments/ List comments on seller's products +
Success Response (200 OK)
Returns all comments on products owned by the authenticated user
